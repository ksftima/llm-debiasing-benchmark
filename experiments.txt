## Goal
Evaluate whether DSL and PPI can replace expensive expert annotations with cheap LLM annotations for NLP tasks. LLM labels are biased but cheap; expert labels are unbiased but expensive. DSL and PPI use a small expert sample + all LLM labels to get the best of both.

## Datasets
- Binary: CUAD, Misogynistic
- On hold (multinomial): PubMedQA, FOMC
- LLMs: llama, deepseek, gpt54, mistral
- Feature variance config: thesis/lib/experiments/dataset_config.json

## Experimental Design
Four phases, progressively more complex downstream model:

| Phase | Model | Estimand |
|-------|-------|----------|
| 1 | Prevalence (intercept only) | θ = P(Y=1) — scalar |
| 2 | Logistic + low-variance feature | θ = [β₀, β_low] |
| 3 | Logistic + high-variance feature | θ = [β₀, β_high] |
| 4 | Full logistic regression | θ = [β₀, β₁, ..., β₅] |

Two experiments:
- Experiment 1: Fix total N, vary expert sample n — "How many expert labels do I need?"
- Experiment 2: Fix n, vary N — "How much LLM data do I need?" (not started)

## Evaluation Metrics
- sRMSE = sqrt(E[((θ - θ*)/θ*)²]) — standardized % error from ground truth
- Standardized Bias = E[(θ - θ*)/θ*] — ≈0 for DSL/PPI (unbiased by design)
- sRMSE_eucl (Phase 2+) = Euclidean norm version for coefficient vectors, used for cross-phase comparison
- SE for CI bands = std(normalized) / sqrt(n_reps) — plotted as ±2 SE shaded bands
- θ* = logistic regression on all N expert labels (ground truth)
- All metrics over 300 repetitions
- n_valid column in CSVs tracks how many reps contributed (after NaN exclusions)

## DSL/PPI Separation Issue (document in thesis)
At small n, both DSL and PPI can fail with singular matrix (complete separation in logistic regression).
- Fix: try/except → [nan, nan]; evaluate uses np.nanmean/nanstd + n_valid column
- CUAD: only DSL fails, ~5/300 reps at n=20-26
- Misogynistic: both fail, ~30 DSL + 17 PPI per 300 reps at n=20-33
- Misogynistic more affected because x2 correlation with y is near zero (r=-0.052), making β*₂ ≈ -0.11 (small denominator amplifies errors)
- Plot filter: n > 50 for Phase 2/3
